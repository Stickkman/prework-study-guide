HTML

  • Basic building block of websites, handles content and structure/framework of the website.
  • The head element contains metadata that aids in rendering the page, usually information that is not seen by user.
  • The body element contains information that we want the user to see and interact with. The body element is a semantic element.
  • Common elements and what they do:
    • 'div' - creates a division in the page
    • 'span' - creates an inline division
    • 'h1' - used for headings/size of text, h1-h6 with size decending, only use one h1 heading per page.
    • 'p' - for paragraphs or blocks of text, used extensively to wrap most text on webpages
    • 'ul' - unordered lists
    • 'ol' - ordered lists
    • 'li' - list items
    • 'br' - line break, creates a line of empty space
    • 'header' - appears at top of the page and usually contains a navigation menu or intro content
    • 'footer' - appears at the bottom of the page, usually ocntains author, copyright, contact info, site map, etc
    • 'section' - semantic elements that define a section in a document that contains thematic content

CSS

  • Stands for Cascading Style Sheets, used to give content style.
  • 'margin' - indicates how much space we want around the OUTSIDE of an element
  • 'padding' - indicates how much space we want around the content INSIDE an element
  • Can use '*' to mark all elements instead of a specific one.
  • Can use the 'class' attribute to share a rule with any element in that class.
  • 3 ways to style a page using CSS:
    • Inline CSS - Elements are styled directly in the HTML files, applies to a particular page element only. Useful for quick and permanent changes, but less flexible than external and internal style sheets.
    • Internal CSS style sheet - Embeds an entire style sheet directly in the HTML file using a style element in the head, rules only apply to that page.
    • External CSS style sheet - Most common way to style a webpage. Must link the file using the 'link' element.

GIT

  • A free open source version control system that enables web developers to collaborate on projects without overwriting each other's work.
  • Allows developers to revert to previous version of code
  • Used with Gitbash, which allows users to use standard unix commands that they would not be able to use otherwise
  • Common commands:
    • 'git status' - checks what branch we are currently on
    • 'git checkout' - changes to specified branch
    • 'git checkout -b' - creates a new branch and switches to it
    • 'pwd' - print working directory, shows current working directory
    • 'ls' - list, shows all files and folder in current directory
    • 'cd' - change directory, (cd ..) goes back a level
    • 'mkdir' - make directory
    • 'touch' - creates a file
    • 'start' - opens a file with default app
    • 'git pull origin main' - *in case local directory is not up to date with github*
  • Always remember to create a new branch off of the main branch before adding or editing code.
  • Basic Workflow process:
    • Create a branch from most recent version of the main branch.
    • Add commits including brief text describing what was done.
    • Open PR, short for Pull Request, content changes are reviewed by team.
    • Merge and Deploy, content changes are approved and publish.

JAVASCRIPT

  • Dynamic high level programming language that allows users to interact with websites. Uses include adding effects, interactivity, highlighting clickable items, etc.
  • A variable is a named container that allows us to store data in our code.
  • Variables in JavaScript are not directly associated with any particular value type, any variable can be assigned and re-assigned values. Example:
    • foo = 42 // foo is now a number.
    • foo = "hello" // foo is now a string.
    • foo = true; // foo is now a boolean.
  • An array is a single variable that is used to hold a group of data, typically related. Can store different types of data at once.
  • To work with individual items inside an array, we need to have the location within the array, referred to as an index.
  • The first item in any array has an index of '0' rather than '1'.
  • We can find the length of an array by using it's built-in ''.length' property added to the end of the array's name. Useful for 'for' loops.
  • A 'for' loop contains 3 important statements:
    • The first determines the starting point for our loop.
    • The second is the condition that keeps the loop running.
    • The third allows the array to iterate over each item. Using 'x++' is an example of this to add 1 to a variable of x to keep count.
  • The .js file must be linked in the html file near the bottom of the page, because you want the elements it will be manipulating to load first.
  • We can comment out multiple lines of code opening with '/*' and ending with '*/'. // can be used for a single line.
  • Control flow is the order in which a computer executes code in a script.
  • Code flows from top to bottom in JavaScript.
  • JavaScript is CASE SENSITIVE, ex. 'myVariable' does not equal 'myvariable'
  • A function is a set of instructions that tells the computer how to perform a certain task. Useful for code we want to reuse.
  • Functions do not execute automatically, they must be called by their unique name.
  • Chrome DevTools is useful when debugging, can be accessed by right clicking > inspect > console. Also with CTRL+SHIFT+I.
